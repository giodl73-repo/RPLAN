# RPLAN Method-Produced Packages

This directory is reserved for small RPLAN packages generated by real BISECT
CLI or crate workflows.

These packages sit between the tiny golden corpus and future benchmark
artifacts:

- tiny golden packages prove that each family can land on the shared verifier
  contract
- method-produced packages prove that a real workflow can generate the package
  shape with stable transcripts and hashes
- benchmark packages, later, can carry larger external-data runs and timing
  claims

## Package Shape

Each package should contain:

```text
package-name/
  plan.rplan
  context.rctx
  audit-certificate.json
  manifest.json
  method-transcript.json
  command-transcript.txt        # optional
  solve-report.json             # optional
  search-report.json            # optional
```

Required package roles are still `plan`, `context`, and `certificate`.
Method-produced packages should also include a transcript role so readers can
see how the package was generated.

## Claim Boundary

A method-produced package proves that:

- the committed files match the manifest hashes
- the audit certificate verifies against the committed RPLAN and RCTX files
- the method lineage and transcript name the generating family, method,
  parameters, seed, and source hashes
- both public verifier surfaces accept the package when applicable

It does not prove large-instance performance, legal sufficiency, fairness,
optimality, or benchmark superiority.

## Verifier Commands

The neutral verifier surface checks the certificate, plan, and context:

```powershell
cargo run -p rplan-cli -- verify-certificate `
  --certificate docs/examples/rplan-method-packages/<package>/audit-certificate.json `
  --plan docs/examples/rplan-method-packages/<package>/plan.rplan `
  --context docs/examples/rplan-method-packages/<package>/context.rctx
```

The BISECT bridge checks the package manifest and then delegates to the RPLAN
certificate verifier:

```powershell
cargo run -p bisect-cli -- verify `
  --manifest docs/examples/rplan-method-packages/<package>/manifest.json
```

## Naming

Use paper-code plus method and source tier:

```text
T.14+spectral-generated-synthetic/
U.18+local-search-generated-descendant/
```

Keep package names stable once committed because papers and docs may link to
them.
