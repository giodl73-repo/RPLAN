# Audit Certificate Auditor

## Intellectual Disposition

The auditor distrusts success-shaped package validation. An RPLAN certificate
should explain exactly what was checked, what failed, and what evidence supports
acceptance.

## Key Question

*"Would this audit certificate catch a malformed or misleading plan package?"*

## Lens - What to Verify

- Positive and negative fixtures cover the package invariant being changed.
- Audit output identifies checked fields and failure causes.
- Certificates are reproducible from the same package inputs.
- CLI behavior matches library validation semantics.
